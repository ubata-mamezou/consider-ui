# ボタン

ボタンは、主にアクション実行またはページ遷移のためのトリガーとして使用します。
画面内におけるボタンの重要度に応じて使い分け可能な複数のスタイルがあります。

## 仕様

### サイズ

高さは規定のサイズを使用してください。
幅はボタンラベルの長さによって可変可とします。デフォルトのサイズより小さいサイズは使用しないでください。

<img src={ButtonSize} alt="サイズ"/>

## 使い方

### スタイル

* プライマリーボタン（重要度：高）
  * 画面内で最も重要なアクションに使用します。
  * ひとつの画面につき、原則1つまで使用できます。
* セカンダリーボタン（重要度：中）
  * プライマリー以外の選択肢がある場合に使用します。
  * 原則として最大で同時に3つまで使用できるものとします。プライマリー以外の選択肢が4つ以上になってしまう時は、フロー自体の設計を見直すことを検討してください。
* ターシャリーボタン（重要度：低）
  * フローやアクションの実行を中断・中止するような、キャンセル行為のために使用します。
  * ひとつの画面につき、原則1つまで使用できます。

<img src={ButtonStyle} alt="スタイル"/>

### 配置

* プライマリーボタン
  * 基本、フッターの右側に配置します。
* セカンダリーボタン
  * フッターに配置する場合は、プライマリーボタンの左隣に配置します。
  * ヘッダーに配置する場合は、右側に配置します。
  * コンテンツエリアに配置する場合は、アクションの内容と関係が深い要素の右隣に配置します。
* ターシャリーボタン
  * フッターに配置する場合は、左側に配置します。
